# كارلي — وكيل قطع الغيار الذكي

نظام بحث عن قطع غيار السيارات عبر الواتساب: العميل/الورشة يرسل طلبًا (قطعة أو أكثر)،
النظام يبثّه للموردين المتخصصين فقط، يجمع الأسعار، ويعرض أفضل عرض لكل قطعة.

## التشغيل المحلي

```bash
npm install
ADMIN_PASSWORD=كلمة_سرك WA_PROVIDER=mock node server.js
```

- صفحة العميل: http://localhost:3000/
- لوحة التحكم: http://localhost:3000/admin.html

في وضع `mock` تُطبع رسائل الواتساب في السجل بدل إرسالها فعليًا — مثالي للاختبار.

## متغيرات البيئة

| المتغير | الوصف | افتراضي |
|---|---|---|
| `ADMIN_PASSWORD` | كلمة مرور لوحة التحكم | `carly2025` (غيّرها فورًا) |
| `WA_PROVIDER` | `mock` للاختبار، `meta` للإرسال الفعلي | `mock` |
| `WA_META_TOKEN` | توكن WhatsApp Business API | — |
| `WA_META_PHONE_ID` | معرّف رقم الواتساب في Meta | — |
| `EMAIL_PROVIDER` | `mock` أو `gmail` للنسخ الاحتياطي | `mock` |
| `GMAIL_USER` | إيميل Gmail المُرسِل | — |
| `GMAIL_APP_PASSWORD` | App Password (ليس كلمة المرور الأساسية) | — |
| `BACKUP_EMAIL` | وجهة النسخ الاحتياطية | carllymotors@gmail.com |
| `PORT` | منفذ الخادم | `3000` |
| `DB_PATH` | مسار قاعدة البيانات | `./carly.db` |

## الصفحات الثلاث

- صفحة العميل `/` — طلب متعدد القطع، عداد 90ث، خيارا الاختيار (مورد واحد / موزّع)
- لوحة التحكم `/admin.html` — الموردون، التقارير، النسخ الاحتياطي، الإعدادات
- لوحة المورد `/supplier.html` — دخول OTP عبر واتساب، الرصيد، سجل الصفقات

## إعداد إيميل النسخ الاحتياطي (عند النشر فقط)

1. فعّل المصادقة الثنائية على Gmail.
2. أنشئ App Password (16 حرفًا) من إعدادات أمان Google.
3. على Render اضبط: `EMAIL_PROVIDER=gmail` و`GMAIL_USER` و`GMAIL_APP_PASSWORD`.
   لا تكتب App Password في الكود — فقط في متغيرات البيئة.

## التقارير (في لوحة التحكم، قابلة للفلترة وتصدير CSV)

أكثر الماركات والقطع طلبًا، توزيع نوع القطعة، أنشط العملاء، أداء الموردين،
الإيرادات الشهرية، وقمع التحويل (طلب → عرض → اختيار).

## النشر على Render

1. ارفع المشروع على GitHub.
2. في Render: New → Web Service → اربط المستودع.
3. Build Command: `npm install`
4. Start Command: `node server.js`
5. أضف متغيرات البيئة أعلاه (خاصة `ADMIN_PASSWORD` و`WA_PROVIDER`).
6. للحفاظ على قاعدة البيانات بين عمليات إعادة التشغيل، أضف Persistent Disk
   ووجّه `DB_PATH` إلى مساره (مثل `/data/carly.db`).
7. استخدم UptimeRobot لإرسال ping كل 5 دقائق يمنع الخمول.

## ربط واتساب الرسمي (لاحقًا)

النظام جاهز للربط عبر طبقة `whatsapp.js` المجرّدة. عند الحصول على
WhatsApp Business API (عبر Meta أو مزوّد): اضبط `WA_PROVIDER=meta` وأضف
التوكن ومعرّف الرقم. لاستقبال ردود الموردين، اربط webhook المزوّد بمسار
`/api/webhook/reply` (الهيكل جاهز، يحتاج محلّل السعر).

## الإعدادات القابلة للتعديل (من اللوحة)

كل هذه تُعدّل من تبويب "الإعدادات" دون لمس الكود:
- عدد الطلبات المجانية لكل مورد (افتراضي 2)
- رسوم الـ Lead بالدرهم (افتراضي 5)
- سقف الدين قبل إيقاف المورد (افتراضي 50)
- تفعيل/إيقاف مرحلة الإطلاق المجاني
- حد الموردين لكل ماركة (افتراضي 8)
- مدة العداد (افتراضي 90 ثانية)
- أوزان حساب درجة المورد (تقييم/التزام/سرعة)

## البنية

- `db.js` — قاعدة SQLite المدمجة + الإعدادات
- `brands.js` — مرجع الماركات ومناشئها
- `suppliers.js` — منطق الموردين، استنتاج المنشأ، حساب الـ Score، البث المؤهل
- `requests.js` — السلة، القطع، العروض، اختيار الفائز، خصم الـ Lead
- `whatsapp.js` — طبقة إرسال مجرّدة (mock / meta)
- `server.js` — الخادم وواجهات API
- `public/index.html` — صفحة العميل
- `public/admin.html` — لوحة التحكم
